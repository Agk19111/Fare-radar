# Fare Radar v3

A personalized, no-AI-quota flight-deal dashboard and email-alert bot.

## What changed from v2

The old page called an AI model directly from browser JavaScript. That made the scan depend on a chat/API usage window, exposed the architecture to CORS/authentication problems, and could never safely protect an API key. Version 3 moves all scheduled work into GitHub Actions.

The backend:

1. Reads public deal-site RSS feeds every three hours.
2. Filters posts for YYJ, YVR, YXX, BLI, SEA, YYC, YEG and unusually cheap YYZ deals.
3. Writes `deals.json` for the dashboard.
4. Emails newly matched posts.
5. Deploys the updated page to GitHub Pages.

The page itself only loads `deals.json`, so pressing **Refresh feed** does not use Claude, ChatGPT or another model.

## Important limitation

This is a deal-post scanner. It does not query all airlines for live inventory. The buttons on the page open Google Flights and Skyscanner to verify current availability. A true universal scanner needs an approved flight-shopping API.

## Setup

1. Create a new GitHub repository. A public repository works with GitHub Pages on GitHub Free. Do not upload any passwords.
2. Upload the contents of this folder to the repository root. Preserve `.github/workflows/scan.yml` exactly.
3. In the repository, open **Settings → Pages** and choose **GitHub Actions** as the source.
4. Open **Settings → Secrets and variables → Actions** and create:
   - `GMAIL_USER`: the Gmail address used to send alerts.
   - `GMAIL_APP_PASSWORD`: a Google app password, not the normal Gmail password.
   - `TO_EMAIL`: where alerts should be delivered.
5. Open **Actions → Fare Radar scan and deploy → Run workflow**.
6. After the workflow completes, open the Pages address shown in the deployment.
7. Edit `index.html` and replace the **Run backend scan** link with your repository Actions URL.

## Files

- `index.html`: dashboard.
- `scanner.py`: feed scanner and email sender.
- `deals.json`: dashboard data generated by the scanner.
- `seen.json`: deduplication state.
- `.github/workflows/scan.yml`: scheduled scan and Pages deployment.

## Security

Never put Gmail app passwords or airline/API keys into `index.html`, JavaScript, `scanner.py`, or a committed configuration file. Store secrets only in GitHub Actions repository secrets.

## Tuning

Edit the configuration near the top of `scanner.py`:

- `ACTIVE_AIRPORTS`
- `CAD_CAPS`
- `YYZ_MAX_CAD`
- `MAX_POST_AGE_HOURS`
- `FEEDS`

A source feed can change or disappear. The workflow records source errors inside `deals.json` and continues scanning the remaining sources.
